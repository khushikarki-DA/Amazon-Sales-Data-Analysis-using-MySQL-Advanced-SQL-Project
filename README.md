# Amazon-Sales-Data-Analysis-using-MySQL
# Project Overview

This project is an advanced SQL-based data analysis project built on an Amazon-style e-commerce dataset. The objective of the project is to solve real-world business problems using MySQL by analyzing customer behavior, sales performance, inventory management, payment trends, seller performance, and product profitability.

The database consists of multiple interconnected tables such as customers, orders, order_items, products, payments, sellers, inventory, shipping, and category tables. An Entity Relationship (ER) Diagram was also created to understand and represent the relationships between the tables.

## Database Structure

The project uses a relational database designed to simulate an e-commerce platform similar to Amazon.

Main Tables Used
Customers
Orders
Order Items
Products
Payments
Sellers
Inventory
Shipping
Category

These tables are connected using primary and foreign keys to enable detailed business analysis across different operational areas.

## Business Problems Solved
1.Top Selling Products Analysis

Analyzed the top-performing products based on total sales value and quantity sold. This helps identify high-demand products and revenue-driving items.

Key Insights:
Calculated total sales value using quantity and price per unit
Identified best-selling products
Compared product-wise sales performance

2.Revenue Contribution by Category

Calculated total revenue generated by each product category and measured its percentage contribution to overall sales.

Key Insights:
Identified high-revenue product categories
Measured category-wise business contribution
Performed revenue share analysis

3.Average Order Value (AOV)

Computed the average order value for customers having more than five orders.

Key Insights:
Identified high-value customers
Analyzed customer purchasing behavior
Evaluated customer spending patterns

4.Monthly Sales Trend Analysis

Analyzed monthly sales trends over the past year using window functions.

Key Insights:
Compared current month sales with previous month sales
Tracked business growth trends
Performed time-series sales analysis using the LAG() function

5.Customers with No Purchases

Identified registered customers who never placed any orders.

Key Insights:
Detected inactive users
Useful for targeted marketing campaigns
Performed customer engagement analysis

6.Least-Selling Categories by State

Determined the lowest-performing product category in each state.

Key Insights:
Performed region-wise sales analysis
Identified weak-performing categories
Used ranking functions for state-level comparisons

7.Customer Lifetime Value (CLTV)

Calculated the total lifetime value generated by each customer and ranked customers accordingly.

Key Insights:
Identified loyal and high-value customers
Performed customer segmentation
Used DENSE_RANK() for customer ranking

8.Inventory Stock Alerts

Queried products with low inventory levels to identify items that require restocking.

Key Insights:
Monitored stock availability
Tracked warehouse-level inventory
Improved inventory management process

9.Inventory Restock Optimization

Predicted products likely to go out of stock soon by comparing current stock with sales velocity.

Key Insights:
Calculated stock-to-sales ratio
Identified fast-moving products
Improved supply chain planning

10.Payment Success Rate Analysis

Calculated the percentage of successful, pending, and failed payments.

Key Insights:
Evaluated transaction success rates
Analyzed payment reliability
Measured payment processing performance

11.Top Performing Sellers

Identified the top sellers based on total sales revenue and analyzed their successful and cancelled orders.

Key Insights:
Compared seller performance
Calculated seller success percentage
Evaluated seller reliability and contribution

12.Product Profit Margin Analysis

Calculated profit margins for products by comparing revenue with cost of goods sold (COGS).

Key Insights:
Identified highly profitable products
Ranked products based on profitability
Performed margin analysis using aggregate calculations

13.Most Returned Products

Analyzed products with the highest return rates.

Key Insights:
Calculated product return percentages
Identified problematic products
Evaluated customer return behavior

14.Increasing Purchase Trend Detection

Identified customers whose order values increased continuously over their last three purchases.

Key Insights:
Tracked growing customer engagement
Detected increasing spending patterns
Used multiple LAG() functions for trend analysis

15.Inactive Sellers Analysis

Identified sellers who had not made any sales in the last six months.

Key Insights:
Tracked inactive sellers
Evaluated seller contribution over time
Analyzed last sales activity

16.Customer Classification

Categorized customers as Returning or New based on their return history.

Key Insights:
Used CASE statements for segmentation
Measured customer return behavior
Performed customer classification analysis

17.Top Customers by State

Identified the top 5 customers in each state based on order volume and total sales.

Key Insights:
Performed state-wise customer analysis
Ranked customers using window functions
Identified region-wise high-value customers

## Conclusion

This Amazon Sales Data Analysis project provides an end-to-end understanding of how SQL can be used to analyze large-scale e-commerce operations. By solving 20 real-world business use cases and implementing advanced SQL concepts, the project demonstrates practical data analytics skills applicable to real-world business environments and data analyst roles.
